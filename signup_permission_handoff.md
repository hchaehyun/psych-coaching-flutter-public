# 회원가입 권한 안내/요청 핸드오프

작성일: 2026-03-13

## 목적

회원가입(온보딩) 과정에 권한 안내 페이지를 추가하는 작업을 다른 Codex 환경에서 이어서 진행할 수 있도록, 지금까지의 논의와 현재 코드베이스 상태를 정리한 문서다.

## 사용자 요청 요약

- 회원가입 시 권한 취득 페이지를 추가하고 싶다.
- 필요하다고 생각한 항목은 `전화`, `사진 촬영`, `사진 접근`이다.
- 초안 이미지는 `권한 안내 화면 -> 전화 권한 팝업 -> 카메라 권한 팝업 -> 앨범 권한 팝업` 흐름으로 보인다.
- 사용자는 "기능을 처음 쓸 때 권한을 받는 대신, 회원가입 초반에 한 번에 받는 것이 많이 어려운지"를 물었다.

## 논의 결과 요약

### 1. 기술 난이도

- `카메라`, `사진 접근`을 회원가입 초반에 미리 받는 것은 기술적으로 어렵지 않다.
- 현재 앱은 이미 단계형 온보딩 구조를 가지고 있어서, 권한 안내 단계를 추가하는 방식이 자연스럽다.

### 2. iOS 제약 사항

- `전화`는 iPhone에서 별도의 런타임 권한 팝업이 없다.
- 따라서 초안처럼 `전화 권한 허용` 시스템 팝업을 띄우는 것은 구현할 수 없다.
- 전화는 권한 요청 항목이 아니라, 필요 시 `tel:` 스킴으로 전화 앱을 여는 동작으로 처리해야 한다.

### 3. UX 권장 방향

- 권한 안내 페이지 자체를 넣는 것은 괜찮다.
- 다만 실제 시스템 권한 요청은 화면 진입 즉시 자동으로 띄우기보다, 사용자가 버튼을 눌렀을 때 띄우는 것이 더 자연스럽다.
- 권한을 거절해도 회원가입 자체는 계속 진행 가능하게 두는 것이 좋다.
- 이미 거절된 권한은 다시 시스템 팝업을 띄울 수 없으므로, 그 경우에는 `설정으로 이동` 버튼을 제공하는 방식이 적합하다.

## 초안 목업에 대한 정리

초안의 방향은 크게 유지할 수 있지만, 아래처럼 수정이 필요하다.

- `전화`
  - "필수 접근 권한"처럼 보이더라도 iOS 시스템 권한 팝업은 없음
  - 화면에는 "위기 상황 시 전화 앱으로 연결하기 위해 사용됩니다" 같은 안내만 표시
- `카메라`
  - 회원가입 중 버튼 탭 시 시스템 권한 팝업 요청 가능
- `앨범(사진 접근)`
  - 회원가입 중 버튼 탭 시 시스템 권한 팝업 요청 가능

즉, 실제 흐름은 아래처럼 잡는 것이 안전하다.

1. 권한 안내 페이지 노출
2. `카메라 허용` 버튼 탭
3. iOS 카메라 권한 팝업 노출
4. `사진 접근 허용` 버튼 탭
5. iOS 사진 권한 팝업 노출
6. `전화`는 안내만 하고 별도 시스템 팝업 없음
7. 거절 상태면 가입은 진행시키고, 나중에 설정 이동 유도

## 현재 코드베이스 확인 결과

### 온보딩 구조

- 파일: `lib/features/onboarding/presentation/pages/onboarding_page.dart`
- 현재 온보딩은 총 4단계다.
  - 0: 약관 동의
  - 1: 닉네임 설정
  - 2: 알림 시간 설정
  - 3: 완료 화면
- 따라서 `권한 안내` 단계를 새로 추가하기 쉬운 상태다.

### 전화 관련 구현

- 파일: `lib/features/home/presentation/pages/home_page.dart`
- `_callEmergency()`에서 `tel:` 스킴으로 전화 앱을 여는 형태다.
- 즉 현재 앱도 전화 권한을 별도로 요청하는 구조가 아니다.

### 사진 관련 구현

- `image_picker`가 이미 `pubspec.yaml`에 포함되어 있다.
- 파일: `lib/features/journal/presentation/pages/journal_page.dart`
- `pickMultiImage()`를 사용하고 있어 사진 접근 관련 기능은 이미 일부 들어가 있다.

### iOS 설정 상태

- 파일: `ios/Runner/Info.plist`
- 현재 `NSPhotoLibraryUsageDescription`는 존재한다.
- 하지만 `NSCameraUsageDescription`는 아직 없다.
- 회원가입에서 카메라 권한까지 요청하려면 `NSCameraUsageDescription` 추가가 필요하다.

### 현재 없는 것

- `permission_handler` 패키지는 아직 의존성에 없다.
- 권한 상태 확인, 거절 상태 분기, 설정 이동까지 다루려면 `permission_handler`를 추가하는 것이 가장 깔끔하다.

## 추천 구현 방향

### 권한 화면 구성

- 온보딩 1단계를 `권한 안내`로 추가
- 각 권한 항목을 카드/리스트 형태로 보여주기
- 상태값 예시
  - `대기`
  - `허용됨`
  - `거절됨`
  - `설정 필요`

### 버튼 동작

- `카메라 허용`
  - `permission_handler`로 카메라 권한 요청
- `사진 접근 허용`
  - `permission_handler`로 사진 권한 요청
  - iOS의 `limited` 권한도 사용 가능 상태로 취급하는 것이 좋음
- `전화`
  - 시스템 권한 요청 버튼 대신 설명 텍스트만 제공

### 예외 처리

- 최초 거절 후:
  - 재요청 가능한 상태면 다시 요청
- 영구 거절/설정 필요 상태면:
  - `openAppSettings()`로 설정 앱 이동 버튼 제공
- 권한이 없어도 가입은 계속 가능하게 처리

## 다른 Codex가 이어서 작업할 때 해야 할 일

1. `permission_handler` 의존성 추가
2. iOS `Info.plist`에 `NSCameraUsageDescription` 추가
3. 온보딩 단계 수를 4 -> 5로 조정
4. `권한 안내` 스텝 UI 추가
5. 카메라/사진 권한 요청 로직 구현
6. 거절/영구거절 시 설정 이동 분기 구현
7. `전화` 항목 문구를 시스템 권한처럼 보이지 않게 수정
8. 기존 온보딩 버튼 라벨과 진행 조건 재조정

## 권장 문구 초안

### 권한 안내 상단 설명

작은결은 서비스 이용에 꼭 필요한 권한과 선택 권한을 안내합니다.  
카메라와 사진 접근 권한은 관련 기능을 사용할 때 필요하며, 허용하지 않아도 기본 기능은 이용할 수 있습니다.  
권한은 언제든지 기기 설정에서 변경할 수 있습니다.

### 전화 항목 설명

위기 상황 시 전화 앱으로 빠르게 연결하기 위해 사용됩니다.

### 카메라 항목 설명

실시간 사진 촬영이 필요한 기능에서 사용됩니다.

### 사진 접근 항목 설명

기기에 저장된 사진을 첨부하거나 불러올 때 사용됩니다.

## 구현 시 주의 메모

- iOS에서는 전화 권한 팝업 구현 불가
- `사진 접근`은 iOS에서 `limited` 상태를 고려해야 함
- 화면 진입 즉시 여러 팝업을 연속 호출하는 것보다, 사용자의 탭 액션마다 하나씩 요청하는 편이 더 안전함
- 회원가입 단계에서 미리 받더라도, 추후 실제 기능 진입 시 권한 상태를 다시 확인하는 방어 로직은 남겨두는 것이 좋음
